# Dashboard 1 — Restaurant Operations

Operational dashboard for a full-service restaurant tracking weekly and monthly performance across revenue, cost control, and table utilization.

## Pages

| Page | Purpose |
|---|---|
| Overview | Headline KPIs + revenue trend + covers by day of week |
| Cost Analysis | COGS% and labor% trends, food vs beverage split, cost breakdown |
| Table Performance | Table turns by day, covers by day, revenue per cover trend |

## Key metrics

| Metric | Definition |
|---|---|
| **Covers** | Guests served per day |
| **COGS%** | (Food cost + Beverage cost) / Total revenue |
| **Labor%** | Labor cost / Total revenue |
| **Table Turns** | Covers / Seats (60-seat venue) |
| **Revenue per Cover** | Total revenue / Total covers |

## DAX measures

All percentage and ratio KPIs use `DIVIDE()` measures rather than averaging raw columns — prevents skew from low-cover days:

```dax
Avg COGS%             = DIVIDE(SUM(restaurant_ops[total_cogs]), SUM(restaurant_ops[revenue]))
Avg Labor%            = DIVIDE(SUM(restaurant_ops[labor_cost]), SUM(restaurant_ops[revenue]))
Avg Table Turns       = DIVIDE(SUM(restaurant_ops[covers]), SUM(restaurant_ops[seats]))
Avg Revenue per Cover = DIVIDE(SUM(restaurant_ops[revenue]), SUM(restaurant_ops[covers]))
Total Revenue         = SUM(restaurant_ops[revenue])
Total Covers          = SUM(restaurant_ops[covers])
```

## Data source

`data/restaurant_ops.csv` — synthetic full-year (2025) daily data generated by `data/generate_restaurant_data.py`. Patterns reflect realistic restaurant operations: weekend lift, monthly seasonality, correlated cost ratios.

To regenerate:
```powershell
uv run data/generate_restaurant_data.py
```
